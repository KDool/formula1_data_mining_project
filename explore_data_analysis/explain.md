# Explanation of EDA Result Figures

The figures generated by `f1_eda.ipynb` summarize the exploratory analysis of
`dataset/outputs/prediction.csv`. They help identify class imbalance, feature
distributions, feature separability, correlation between predictors, and the
most important variables for a baseline Random Forest model.

## 1. Target Variable Distribution

Figure: `eda_01_target_distribution.png`

This plot shows the distribution of the prediction target:

- `podium`: 366 rows
- `points`: 854 rows
- `no_points`: 1234 rows

The dataset is imbalanced. The `no_points` class appears most often, while
`podium` is the smallest class. This means raw accuracy can be misleading,
because a model can perform reasonably well by over-predicting the majority
class.

For modeling, metrics such as macro F1-score, balanced accuracy, and per-class
precision/recall are more informative than plain accuracy. Models such as
Random Forest or SVM should also consider `class_weight="balanced"` so that
errors on the smaller classes are penalized more strongly.

## 2. Feature Distributions

Figure: `eda_02_feature_distributions.png`

This figure shows detailed histograms for six important predictors. The updated
plot uses more bins, larger subplots, denser x-axis ticks, grid lines, and a red
dashed median line for each feature. Some plots also show minimum/maximum
values and note when values fall outside the displayed range. This makes the
main distribution easier to read without letting rare extreme values compress
the entire plot.

### `driver_age`

Most drivers are concentrated between their mid-20s and early-30s, with fewer
older drivers. Age may capture experience or career stage, but it is probably
less important than car performance, qualifying, and recent race form.

### `days_since_last_race`

Most races happen close together, especially around 7-14 day gaps, but there
are some large gaps. The plot uses a focused x-axis so normal race-calendar
spacing is easier to inspect. The value `-1` is a sentinel for no previous race
history, not a real negative number of days.

Large values can correspond to season breaks, missed races, driver absences, or
calendar interruptions. This feature can be useful because recent form may be
less reliable after a long break.

### `points_gap_to_leader`

This feature is strongly right-skewed. Many drivers have smaller gaps, while a
smaller number have very large gaps to the championship leader. For
distance-based models such as SVM, this feature should be scaled. A robust
scaling method is preferable because the distribution contains large values.
The `-1` value is a sentinel for missing/previous-standing history, so it should
be handled carefully before scaling.

### `driver_avg_position_last3`

This feature measures recent finishing form over the previous three races.
Lower values are better because position 1 is the best result. The distribution
covers a wide range, making it useful for separating front-running drivers from
midfield and backmarker drivers. The `-1` value is a cold-start sentinel for no
available history.

### `teammate_h2h_avg_position_delta`

This feature is centered near zero. Negative values mean the driver usually
finished ahead of their teammate, while positive values mean the driver usually
finished behind. It contains useful information, but it is not enough by itself
because beating a teammate in a slow car is different from beating a teammate in
a top car.

### `driver_dnf_rate_historical`

This feature describes the historical rate of non-finishes. It may capture
reliability or risk, but it can also reflect team reliability rather than only
driver behavior. The normal DNF-rate range is 0 to 1, while `-1` is a sentinel
for drivers without prior history.

## 3. Separability: Recent Average Position

Figure: `eda_03_separability_driver_avg_position_last3.png`

This boxplot checks whether `driver_avg_position_last3` separates the target
classes.

The feature separates the classes very well:

- `podium` drivers usually have the lowest recent average finishing positions.
- `points` drivers are generally in the middle.
- `no_points` drivers usually have the highest recent average finishing
  positions.

This is one of the strongest predictors because it captures both recent driver
form and car competitiveness. Since lower finishing positions are better, the
model can use this feature to identify drivers who are likely to finish near the
front.

## 4. Separability: Teammate Head-to-Head Delta

Figure: `eda_03_separability_teammate_h2h_avg_position_delta.png`

This boxplot checks whether `teammate_h2h_avg_position_delta` separates the
target classes.

There is some signal:

- `podium` drivers tend to have slightly negative deltas, meaning they often
  outperform their teammate.
- `points` drivers are centered close to zero.
- `no_points` drivers tend to be slightly higher.

However, the boxplots overlap heavily. This means the feature is not very strong
on its own. It becomes more useful when combined with constructor strength,
qualifying position, grid position, and recent race form.

## 5. Spearman Correlation Heatmap

Figure: `eda_04_correlation_heatmap.png`

This heatmap shows Spearman rank correlation between numeric predictors.
Spearman correlation is appropriate here because many Formula 1 features are
rank-like, such as finishing position, grid position, and standings position.

Important patterns:

- `grid` and `qualifying_position` are strongly correlated, which is expected
  because grid position is usually determined by qualifying.
- Rolling form features such as `last3`, `last5`, and `last10` versions are
  highly correlated with each other.
- Driver and constructor strength features are correlated because strong
  drivers usually drive strong cars.
- Points average, podium rate, and average position features form a large
  correlated block.

This correlation is important for model choice. Random Forest can handle
correlated predictors reasonably well, although feature importance becomes
harder to interpret. SVM can be more sensitive to redundant dimensions, so
scaling and feature selection may improve performance.

## 6. Random Forest Feature Importance

Figure: `eda_05_feature_importance.png`

This plot shows the top predictors according to Gini importance from a baseline
Random Forest model.

The most important features include:

- `qualifying_position`
- `grid`
- `driver_points_avg_last10`
- `driver_points_avg_last5`
- `driver_avg_position_last10`
- `constructor_avg_position_last5`
- `driver_dnf_rate_historical`
- previous standings features
- `points_gap_to_leader`

The model relies heavily on qualifying/grid position, recent driver form,
constructor performance, and championship context. This matches Formula 1 domain
knowledge: race outcome is strongly influenced by starting position, car pace,
recent form, and team strength.

One caution is that Gini importance can overstate the importance of correlated
features. For example, `grid` and `qualifying_position` measure similar
information, and the rolling form features overlap strongly. Therefore, this
plot should be interpreted as showing important feature families, not a perfect
ranking of individual variables.

## Overall Interpretation

The EDA shows that the dataset contains meaningful predictive structure.
Recent race form, qualifying position, grid position, constructor strength, and
championship standings are the most useful signals. The target classes are
imbalanced, so evaluation should focus on class-aware metrics rather than raw
accuracy. Correlated rolling features are useful for Random Forest, but they may
need scaling, feature selection, or dimensionality reduction before training an
SVM.

## Suggested Modeling Approaches

### Random Forest

Random Forest is the strongest practical candidate for this dataset. The EDA
shows that race outcome depends on several interacting signals: qualifying
position, grid position, recent driver form, constructor strength, standings,
DNF history, and teammate performance. These relationships are not purely
linear. For example, a good teammate head-to-head value is more meaningful when
the driver is also in a strong constructor and starts near the front.

Random Forest is suitable because it can model these non-linear interactions
without requiring manual feature combinations. It also handles skewed variables,
sentinel values such as `-1`, and correlated rolling-window features better than
SVM. The main limitation is interpretability: correlated features such as
`grid` and `qualifying_position`, or `last3`, `last5`, and `last10` rolling
features, can split importance across similar predictors.

Suggested approach:

```python
RandomForestClassifier(
    n_estimators=300,
    class_weight="balanced",
    random_state=42,
    n_jobs=-1
)
```

Use macro F1-score, balanced accuracy, and per-class recall to avoid hiding poor
performance on the smaller `podium` class.

### Decision Tree

Decision Tree is useful as an interpretable baseline. The EDA shows that some
features have strong direct separation, especially `qualifying_position`,
`grid`, and recent average position. A tree can turn these patterns into simple
rules, such as drivers starting near the front with strong recent form being
more likely to finish on the podium.

The main advantage is explainability. A shallow tree can be visualized and used
to understand which features create the first major splits. The main weakness is
overfitting. A deep tree can memorize race-specific patterns, especially because
the dataset contains correlated features and an imbalanced target.

Suggested approach:

```python
DecisionTreeClassifier(
    max_depth=4,
    class_weight="balanced",
    random_state=42
)
```

Use this model as a baseline and explanation tool. If performance is much lower
than Random Forest, that is expected because a single tree has higher variance
and less ability to average out noisy splits.

### SVM

SVM can be used as a comparison model, but it needs the most careful
preprocessing. The EDA shows several challenges for SVM: feature scales are very
different, `points_gap_to_leader` is heavily skewed, some columns use `-1` as a
sentinel value, and many rolling-window features are highly correlated.

An RBF-kernel SVM is more appropriate than a linear SVM because several useful
signals are interaction-based. For example, teammate performance only becomes
strongly meaningful when combined with constructor strength, grid position, and
qualifying position. However, because SVM is distance-based, scaling is
mandatory.

Suggested approach:

```python
Pipeline([
    ("scaler", RobustScaler()),
    ("svm", SVC(kernel="rbf", class_weight="balanced"))
])
```

Before training SVM, consider handling sentinel values explicitly, scaling all
numeric features, and reducing redundant rolling-window features. SVM may be
useful for comparison, but Random Forest is likely easier to train and more
robust for this feature set.
